# 🏎️ Pit Stop - Gestão Automotiva

> **Projeto Integrador I - Ciência da Computação (5º Semestre)** > **Centro Universitário CEUB**

O **Pit Stop** é uma aplicação web responsiva projetada para funcionar como uma "Garagem Digital". O sistema permite que motoristas façam a autogestão inteligente de seus veículos, registrando manutenções preventivas e corretivas, centralizando gastos e recebendo alertas de revisões.

---

## 🎯 Objetivo do Projeto
Atacar a desorganização no controle de manutenção automotiva do motorista comum. O sistema transforma recibos e anotações soltas em um histórico digital valorizado do veículo, ajudando a estender a vida útil das peças, evitar manutenções corretivas emergenciais e aumentar o valor de revenda do automóvel.

## 🛠️ Tecnologias Utilizadas
Este projeto foi construído utilizando as seguintes tecnologias e ferramentas:

* **Front-end e Back-end:** Node.js com framework Express.
* **Linguagem:** JavaScript (ES6).
* **Testes Automatizados (E2E):** Playwright.
* **Gestão e CI/CD:** Azure DevOps (Boards, Repos e Pipelines).
* **Prototipagem:** Figma.

## 🚀 Como executar o projeto localmente

Siga as instruções abaixo para rodar a aplicação na sua máquina:

**1. Clone o repositório:**
\`\`\`bash
git clone <URL_DO_SEU_REPOSITORIO_AZURE>
\`\`\`

**2. Acesse a pasta do projeto:**
\`\`\`bash
cd <NOME_DA_PASTA>
\`\`\`

**3. Instale as dependências do Node:**
\`\`\`bash
npm install
\`\`\`

**4. Inicie o servidor web:**
\`\`\`bash
npm start
\`\`\`
*O aplicativo estará rodando no seu navegador em: `http://localhost:3000`*

## 🧪 Como rodar os testes automatizados
O projeto conta com uma suíte de testes ponta a ponta (E2E) desenvolvida com Playwright para validar os fluxos principais (ex: Login válido e inválido).

Para executar os testes, utilize o comando:
\`\`\`bash
npm test
\`\`\`
